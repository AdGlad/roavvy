# Tasks

Active and upcoming work items for Roavvy.

## Format

Each task file (or section) should include:

- **Goal** — what the user can do after this work
- **Scope** — in / out
- **Acceptance criteria** — testable conditions for done
- **Assigned persona** — Planner / Architect / Builder / Reviewer / UX Designer
- **Status** — `backlog` | `in progress` | `in review` | `done`

## Conventions

- Tasks are named `NNN-short-description.md` (e.g. `001-initial-scan-flow.md`).
- Completed tasks move to `tasks/archive/`.
- The Planner persona is responsible for keeping this folder current.
