# M141 — Merch Cart Gallery View & Order History Thumbnails

## Goal

Replace the `ListTile`-based rendering in the cart and order history with visual gallery
displays. Cart items should feel like saved designs worth buying; order history should
feel like a permanent collection of travel artefacts.

---

## Phases & Tasks

### T1 — Cart item gallery card widget

**File:** `apps/mobile_flutter/lib/features/merch/merch_shop_screen.dart`

Replace `_ShopCartItemTile` (which uses `ListTile` with a 56×56 thumbnail) with a new
`_ShopCartItemCard` widget that renders as a visual card:

```
┌────────────────────────────────────┐
│  [shirt mockup — full width, 200px]│  ← frontMockupUrl or local placeholder
├────────────────────────────────────┤
│  The Grand Tour                    │  ← MerchCartItem.title
│  Roavvy: 42 countries              │  ← subtitle from cart item
│  Black · L                         │  ← colour + size
│                    [Ready ✓]       │  ← status badge
│  [Delete]          [Checkout →]    │  ← actions
└────────────────────────────────────┘
```

Implementation notes:
- Mockup image: use `frontMockupUrl` from `MerchCartItem` if non-null and non-empty.
  Fallback: a styled placeholder container with the shirt icon and the design title
  centred (do not show a blank grey box).
- Design title: `MerchCartItem.title` — display name generated by `MerchStory` and
  stored at cart creation time (ensured by M137-T1).
- Subtitle: derive from stored title or fall back to "X countries".
- Status badge: keep existing `_StatusBadge` widget, unchanged.
- Checkout CTA: tapping the card or the "Checkout →" button opens
  `CartItemCheckoutScreen` — same as current `onTap`.
- Delete: IconButton with `Icons.delete_outline`, same confirmation dialog.
- Swipe-to-delete: `Dismissible` wrapper with red background — additive.
- Arrange cards in a `ListView` with 12px vertical padding between cards (not
  `Divider(height: 1)` as currently).

### T2 — Order history thumbnail + story title

**File:** `apps/mobile_flutter/lib/features/merch/merch_orders_screen.dart`

Replace `_OrderTile` (`ListTile`) with `_OrderCard`:

```
┌─────────────────────────────────────┐
│  [shirt mockup — 120px left column] │  ← thumbnailUrl (if stored)
│                                     │
│  The Grand Tour                     │  ← designTitle
│  42 countries · 4 Jun 2026          │  ← countryCount + date
│                         [Ordered ✓] │  ← status badge
│                      [Design again] │  ← "Buy Again" relabelled
└─────────────────────────────────────┘
```

Add `designTitle` and `thumbnailUrl` fields to `MerchOrderSummary`:

```dart
class MerchOrderSummary {
  // existing fields ...
  final String? designTitle;     // ADD — from 'title' field in merch_configs doc
  final String? thumbnailUrl;    // ADD — from 'mockupUrl' or 'previewUrl' field
```

Update `MerchOrderSummary.fromDoc` to read these fields:

```dart
final designTitle = data['title'] as String? ?? data['designTitle'] as String?;
final thumbnailUrl = data['mockupUrl'] as String? ?? data['previewUrl'] as String?;
```

These fields are already written to `merch_configs` by the `createMerchCart` Firebase
Function — no backend change needed. Older documents without the field gracefully
degrade (null → no image shown).

The `_OrderCard` thumbnail column:
- 120px wide, `BoxFit.cover` within a `ClipRRect(borderRadius: 8)`.
- If `thumbnailUrl` is null: show a subtle gradient placeholder with country count
  centred (e.g. "42" in large bold white text on a `RoavvyColours.backgroundDark`
  background).

Relabel the "Buy Again" tap action button to "Design again" (it navigates to
`LocalMockupPreviewScreen` — the correct framing is designing, not buying).
